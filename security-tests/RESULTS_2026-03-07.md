### Failures explained

- **is_guest references**: `is_guest` is used as a local/in-memory flag in `players.tsx` and `tees.tsx` to distinguish guest players during round creation. There is no `is_guest` column in the database (confirmed by schema test). This is a code smell but not a security issue — the flag is only used client-side to render a "Guest" badge and build the correct payload for the create-round edge function. **Recommendation**: Rename to a clearer local flag like `isLocalGuest` or derive from `id.startsWith('guest_')` to avoid confusion with the database schema.

### Warnings explained

- **Anon key duplicated in 3 files**: The Supabase anon key appears in `rounds.tsx`, `round-summary/[roundId].tsx`, and `live-scoring/[roundId].tsx` as hardcoded `apikey` headers. The anon key is public by design (it's in the client bundle), but it should be imported from `lib/supabase.ts` instead of duplicated. Not a security vulnerability, but a maintenance concern.

- **AsyncStorage for auth**: Expo's standard approach. SecureStore would be more secure for token storage but requires native modules. Low risk for a golf scoring app.
